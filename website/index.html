Why VoIPC?

Built for people who believe privacy isn't a premium feature — it's a right.

Zero Knowledge

The server is blind by design. It never sees your messages, never stores chat history, never logs conversations. All encryption and decryption happens on your device.

Signal Protocol

The same end-to-end encryption trusted by billions. X3DH key exchange, Double Ratchet algorithm, Perfect Forward Secrecy. Your past messages stay safe even if a key is compromised.

Self-Hosted

Your server, your rules. A single Rust binary with zero disk persistence. Server restart means a clean slate. No accounts, no sign-up, no email — just connect and talk.

Truly Open Source

Every line of code is public under the MIT license. Audit it, fork it, contribute. No hidden telemetry, no phone-home, no data monetization. Ever.

Five Layers of Security

"Encryption is not a feature. It's the foundation."

1

Transport Security

TLS on every connection
  • rustls — pure-Rust TLS implementation with ring crypto provider
  • TLS 1.2+ encrypts all TCP control traffic (auth, channel ops, chat, key exchange)
  • TOFU (Trust-On-First-Use) certificate pinning for self-signed server certs
  • No plaintext TCP connections are accepted — TLS is mandatory
2

End-to-End Message Encryption

Signal Protocol — the gold standard
  • X3DH (Extended Triple Diffie-Hellman) key exchange for session establishment
  • Double Ratchet algorithm provides forward secrecy per message
  • Curve25519 identity keys (32-byte) with Ed25519 signed pre-keys
  • 100 one-time pre-keys per user + 1 signed pre-key, auto-replenished
  • Sender Keys for efficient group/channel message encryption
  • Perfect Forward Secrecy — compromising a key cannot decrypt past messages
  • Uses the official libsignal-protocol crate from Signal Foundation
3

Media Encryption

AES-256-GCM on every voice & video packet
  • AES-256-GCM symmetric encryption with per-channel 256-bit keys
  • Deterministic nonce: session_id(4) || sequence(4) || extra(4) = 12 bytes — mathematically prevents nonce reuse
  • 16-byte authentication tag detects any packet tampering
  • AAD (Additional Authenticated Data) binds channel_id + packet_type, preventing cross-channel replay attacks
  • Mandatory key rotation after ~4.3 billion packets (~24h at 50 pkt/sec)
  • Media keys distributed to channel members encrypted via Signal sessions
4

Local Storage Encryption

Your device, your vault
  • Chat history encrypted with PBKDF2-HMAC-SHA256 (600,000 iterations) + AES-256-GCM
  • 32-byte random salt per file + 12-byte random nonce
  • Custom binary format: VOIP magic header + version + salt + nonce + encrypted payload + GCM tag
  • Signal Protocol state encrypted separately with VSIG file format
  • Max 500 messages per channel stored locally (auto-rotated)
  • Zeroizing<T> for all secrets in memory — overwritten on drop, never leak to swap
5

Zero-Knowledge Server

The server is blind by design
  • Server NEVER sees plaintext messages — encrypted client-side with Signal Protocol
  • Server NEVER stores chat history — zero persistence, no disk writes
  • Server NEVER decodes voice or video — SFU architecture relays encrypted packets
  • Server NEVER logs conversations — memory-only state, restart = clean slate

Run Your Own Server

From zero to encrypted voice chat in under 5 minutes.

1. Build the server
# Clone the repo
git clone https://github.com/your-org/voipc.git
cd voipc

# Build the server binary
cargo build -p voipc-server --release
2. Generate TLS certificates
mkdir -p certs
openssl req -x509 -newkey ec \
  -pkeyopt ec_paramgen_curve:prime256v1 \
  -keyout certs/server.key \
  -out certs/server.crt \
  -days 365 -nodes \
  -subj "/CN=voipc"
3. Run
./target/release/voipc-server

Requirements & Configuration

  • Rust Install via rustup.rs. Stable toolchain.
  • Linux deps libssl-dev, pkg-config, libasound2-dev (for audio)
  • Ports TCP & UDP 9987 (default, configurable)
  • Config TOML config file: tcp_port, udp_port, max_users, cert_path, key_path
  • Settings server_settings.json: empty_channel_timeout, max_channels
